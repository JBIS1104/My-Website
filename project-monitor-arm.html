Monitor Arm.

Overview

Why it exists.

Conventional monitor arms require manual adjustment every time a user changes posture or repositions themselves. This project removes that friction entirely.

An iPad streams live video via MJPEG over HTTPS to a Raspberry Pi 5. The RP5 runs an OpenCV Haar cascade face detector on each incoming frame, computes the pixel offset from frame centre, and drives two servo axes directly via hardware PWM on GPIO pins 18 and 13 — keeping the monitor centred on the user’s face continuously.

Target applications include desk productivity, smart TVs, portable displays for accessibility, and collaborative workspace screens.

Software pipeline

From face to servo.

The system runs a continuous closed-loop pipeline entirely on the Raspberry Pi 5: stream → detect → compute offset → PWM → repeat.

1. MJPEG Stream from iPad

An iPad acts as the camera, streaming live video to the Raspberry Pi 5 via MJPEG over HTTPS. The RP5 parses the JPEG boundary markers (\xff\xd8 / \xff\xd9) from the raw HTTP byte stream, decodes each frame with OpenCV, and feeds it directly into the detection stage.

2. Haar Cascade Face Detection

Each frame is converted to greyscale and processed by an OpenCV Haar cascade classifier (haarcascade_frontalface_default.xml). When multiple faces are present, the largest bounding box by area is selected as the primary target. The centroid of that bounding box is passed to the servo update stage.

3. Dead-band Offset Calculation

The face centroid is compared against the frame centre. A dead-band zone (15% of frame width/height) filters out small movements to prevent servo jitter. Outside the dead-band, the duty cycle is incremented or decremented by a fixed step (0.5%) at 100 ms intervals, clamped between 0% and 100%.

4. Hardware PWM on GPIO 18 & 13

PWM signals are generated directly on GPIO pin 18 (pan/X axis) and GPIO pin 13 (tilt/Y axis) at 400 Hz using gpiozero.PWMOutputDevice with the lgpio pin factory for hardware-accurate timing. On startup, a smooth ramp function moves both servos to their home positions before tracking begins.
